# whirlwind unwrap — current-build report (2026-06-01)

Default pipeline (no Goldstein filter): **reuse solver → tiled-512 + global coarse
anchor + multi-scale cascade + bounded sliver cleanup → gated multi-shift re-solve
→ seam-repair**, memory-bounded, matching SNAPHU/PHASS quality on coherent terrain
at a fraction of the runtime.

## The A_016 fix: gated multi-shift re-solve

A_016 is mostly a large decorrelated body with coherent land threading around it.
At the default tile grid, whirlwind produced a **valid** unwrapping (wrapped
residual ≈ 0) that differed from production by (a) a **wrong global winding** on
the right portion and (b) a **high-coherence seam-strip artifact** at a tile
boundary (a spurious branch-cut loop in the overlap zone). Both are tile-grid
dependent: a grid shifted by half a tile step resolves them (the seam moves; a
different winding is selected).

**Diagnosis (why simpler fixes fail):**
- Whole-image solve is *worse* (45–47%) — bare MCF mis-routes the winding and the
  linear cost runs away; the "single-tile = best" intuition does not hold here.
- Pyramid / coarse anchor mis-levels (fewer edges across the neck ⇒ wrong integer
  is cheaper); region-leveling can't isolate a *circulation* (back-path connected).
- A same-cost secondary cannot recover production: whirlwind's solution is
  *cost-optimal under its coherence cost* — production's smoother winding is
  pricier under that cost (it is preferred only by SNAPHU's statistical cost).

**The fix (a correct unwrap never tears coherent terrain):** run the tile grid;
compute the rate of branch cuts through high-coherence (>0.7) pixels; if it is
significantly nonzero (the artifact signature), re-run on tile grids shifted by
¼, ½, ¾ of the tile step and **return the result with the fewest coherent cuts**.

- A_016 coherent-cut rate ≈ 6.7e-3 (per valid px) ⇒ gate fires ⇒ 54.8 → 96.8%.
- Clean / noisy-fine scenes ≤ 5.6e-4 ⇒ gate **does not fire** ⇒ byte-identical,
  1× cost. Only A_016 pays the ~4× cost (60 s, far under the 5-min bar).
- See `a016_diag/a016_before_after.png`.

**Seam-repair (second stage).** The global shift selection minimizes *total*
coherent cuts, so it can still leave a *local* block — e.g. a coherent land corner
of a water-dominated tile stuck at the wrong cycle at a residual seam. After the
shift is chosen, `seam_repair` finds each large cluster of high-coherence cuts,
re-unwraps a window around it **seam-free** (`unwrap_reuse`), and snaps only the
largest connected single-integer disagreement — and only if that strictly reduces
the window's coherent-cut count (monotonic; cannot regress). This took A_016's
residual water-tile block from 16.1k wrong pixels to 0.5k (96.81 → 97.11%) and is
an exact no-op on the four clean frames (0 pixels snapped).

Implemented as `whirlwind_core::tile::unwrap_tiled_robust` (the default for
auto-tiled frames). Tests: `coherent_cut_rate_*`, `unwrap_tiled_robust_is_noop_*`.

## Honest residuals / known limits

- A_016 tops out near **97%** (matching the best tiled result); the remaining ~3%
  is a genuinely-ambiguous winding through the decorrelated body that neither
  whirlwind's nor production's cost can resolve without extra model assumptions.
- Low-coherence regions everywhere are per-pixel noisy and the production
  reference is unreliable there; report mainland numbers, not full-image.
- The multi-shift gate threshold (`COH_CUT_FLOOR`) is empirically calibrated with
  a >3× margin on the 7 test scenes; broader GUNW coverage would further validate
  it. Matching SNAPHU on *all* fragmented scenes would require porting SNAPHU's
  statistical MAP cost (large; the prior convex/LLR-cost exploration was mixed).
